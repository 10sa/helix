Module ix.storage

Player manipulation of inventories.

This library provides an easy way for players to manipulate other inventories. The only functions that you should need are ix.storage.Open and ix.storage.Close. When opening an inventory as a storage item, it will display both the given inventory and the player's inventory in the player's UI, which allows them to drag items to and from the given inventory.

Example usage:

ix.storage.Open(client, inventory, {
    name = "Filing Cabinet",
    entity = ents.GetByIndex(3),
    bMultipleUsers = true,
    searchText = "Rummaging...",
    searchTime = 4
})

Storage info structure

There are some parameters you can customize when opening an inventory as a storage object with ix.storage.Open.

  • id
    (default: id of inventory passed into ix.storage.Open)
    The ID of the inventory.

  • name
    (default: "Storage")
    Title to display in the UI when the inventory is open.

  • entity
    (required)
    Entity to "attach" the inventory to. This is used to provide a location for the inventory for things like making sure the player doesn't move too far away from the inventory, etc. This can also be a player.

  • bMultipleUsers
    (default: false)
    Whether or not multiple players are allowed to view this inventory at the same time.

  • searchTime
    (default: 0)
    How long the player has to wait before the inventory is opened.

  • searchText
    (default: "@storageSearching")
    Text to display to the user while opening the inventory. This can be a language phrase.

  • OnPlayerClose
    (default: nil)
    Called when a player who was accessing the inventory has closed it. The argument passed is the player who closed it.

Functions

ix.storage.GetReceivers(inventory) Returns all players that currently looking at the given inventory as a storage.
ix.storage.InUse(inventory) Returns whether or not the given inventory has a storage context and is being looked at by other players.
ix.storage.CreateContext(inventory, info) Creates a storage context on the given inventory.
ix.storage.RemoveContext(inventory) Removes a storage context from an inventory if it exists.
ix.storage.Sync(client, inventory) Synchronizes an inventory with a storage context to the given client.
ix.storage.AddReceiver(client, inventory, bDontSync) Adds a receiver to a given inventory with a storage context.
ix.storage.RemoveReceiver(client, inventory, bDontRemove) Removes a storage receiver and removes the context if there are no more receivers.
ix.storage.Open(client, inventory, info) Makes a player open an inventory that they can interact with.
ix.storage.Close(inventory) Forcefully makes clients close this inventory if they have it open.
